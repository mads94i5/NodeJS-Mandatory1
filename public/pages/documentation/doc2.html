Variables, data types

Variables

Variables are used to store data in memory during runtime. In JavaScript, the keyword 'var' was the original way to declare variables. But this is now considered outdated and is not recommended for modern development.
Variables declared with 'var' have function scope or global scope, depending on where they are declared. This can lead to unintended variable hoisting and scope-related issues.
In practice, this means you can use a variable before it is declared, giving the inital value of 'undefined'. Example of 'var' declaration:

var name = "John"

In 2015, ES6 was introduced and in turn also a new way to declare viables in JavaScript.
The new keywords 'let' and 'const' were added. With both having block scope, meaning they are only accessible within the block where they are defined.
Variables declared with 'let' can be reassigned after initialization, but 'const' can not. However, the value 'const' points to, in case of objects and arrays, can be modified.
Examples of 'let' and 'const':

let age = 30;
const pi = 3.14159;

age = 31;

console.log(age); // Output: 31
console.log(pi); // Output: 3.14159

In JavaScript, the naming convention of variables is to use camelCase and variable names are case-sensitive. Descriptive and meaningful naming makes code more readable.

Data types

Data types in JavaScript are loosely typed (also known as dynamically typed), meaning that variable data types are determined at runtime.
JavaScript has type coercion and types are interpreted. So if you add a string with a number, the number becomes a string:

2 + "2" // returns "22"

JavaScript has 7 different primitive data types:

Number: represents both integer and floating-point numbers.
Example:

let num = 42;
let decimalNum = 5.63;

String: represents a sequence of characters. There are 3 different operators to declare a string.
Example:

// If I need single quotes:
let first = "This is the 'first' way";
// If I need double quotes:
let second = 'This is the "second" way';
// If I need both or variable in the middle or need multi-line:
let third = `This is the "${2 + 1}rd' way.
this 
is
multi-line`;

Boolean: represents a binary value, either 'true' or 'false'.
Example:

let isTrue = true;
let isFalse = false;

Undefined: represents a value that has been declared but not assigned a value.
Example:

let undefinedVar;

Null: represents an intentional absense of any object value or value at all.
Example:

let emptyValue = null;

Symbol (ES6+): represents a unique and immutable value.
Example:

const uniqueSymbol = Symbol("description");

BigInt (ES11+): represents large integers that cannot be represented by the 'Number' type.
Example:

const bigIntValue = 1234567890123456789012345678901234567890n;

JavaScript also has 7 different reference data types:

Object: represents a collection of key-value pairs, where keys are strings (or Symbols) and can be of any data type. Objects are created using object literals '{}' or the 'new Object()' constructor.
Example:

let person = {
name: "Alice",
age: 30
};

Array: represents an ordered collection of values, which can be of any data type. Arrays are created using array literals '[]' or the 'new Array()' constructor.
Example:

let numbers = [1, 2, 3, 4, 5];

Date: represents dates and times. Date objects can be create using the 'Date()', 'new Date()' constructors or the static method 'Date.now()'.
When you call 'Date()' without any arguments, it returns a string representing the current date and time in the local time zone.
When 'new Date()' is called, it gives the current date and time in UTC.
When the 'Date.now()' static method is called, it returns the current timestamp as the number of milliseconds since January 1st, 1970. Also known as the Unix epoch or epoch timestamp. Example:

console.log("Date():", Date());
// Output: Date(): Fri Sep 21 2023 14:30:00 GMT+0200 (Central European Summer Time)
console.log("new Date():", new Date());
// Output: new Date(): Fri Sep 23 2023 11:30:00 GMT+0000 (Coordinated Universal Time)
console.log("Date.now():", Date.now());
// Output: Date.now(): 1669216200000

RegExp: represents regular expressions, used for pattern matching.
Example:

let pattern = /abc/;

Map: is a collection for storing key-value pairs.
Example:

// Creating a map:
let map = new Map();
// Adding to a map:
map.set("name", "Alice");
// Accessing values in a map:
console.log(map.get("name")); // Output: "Alice"
// Checking if a key exists in a map:
console.log(map.has("name"));    // Output: true
// Deleting a key-value pair in a map:
map.delete("age");

Set: is a collection for storing unique values. It does not allow duplicate values.
Example:

// Creating a set:
let set = new Set();
// Adding to a set:
set.add("apple");

set.delete("apple");
