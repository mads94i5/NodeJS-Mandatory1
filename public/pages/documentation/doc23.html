Structuring HTML with semantic HTML

It can be benefitial to use semantic HTML to structure your HTML. It makes the code more readable and can help understanding larger elements. They are purely semantic in nature and have no functionality besides organisation. Some of the most common ones are:

<header>
<nav>
<main>
<section>
<article>
<aside>
<footer>

These are used in cases where their name would make sense to group other elements. The <header> element could also contain <nav> elements, as it's common to have navigation on a header. So they can also be nested within each other.
