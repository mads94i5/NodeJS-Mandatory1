package.json

The package.json file is the starting point to setting up an application with the Express framework. It contains the configuration, metadata and dependencies of the project. The dependencies are the most vital.
The file has to be located in the root project folder.
Here is an example of a simple package.json file: 

{
    "name": "pokemon",
    "version": "0.0.1",
    "description": "Pokemon website",
    "type":"module",
    "main": "app.js",
    "scripts": {
        "test": "echo \"Error: no test specified\" && exit 1",
    },
    "keywords": [
        "pokemon",
        "pokemon",
        "battle",
        "website",
        "node",
        "express"
    ],
    "author": "mads94i5",
    "license": "ISC",
    "dependencies": {
        "express": "^4.18.2"
    },
    "devDependencies": {
        "eslint": "^8.49.0"
    }
}

As part of the configuration, you can define scripts and run them in the terminal: 

npm run test

This will run the test script, which in this case is just calling an echo and exiting.
