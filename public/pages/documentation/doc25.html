Environment variables

In Node, you can define environment variables when you run the application. But this is dependent on your operating system and terminal.
For Windows: 

SET PORT=8080 && node app.js

For Mac and Linux: 

PORT=8080 node app.js

To counter this, you can install a package like cross-env. This makes development with different operating systems seamless.
To install cross-env, type: 

npm i cross-env

Now you can set up scripts in your 'package.json' to run the application using the cross-env wrapper:

"scripts": {
    "test": "echo \"Error: no test specified\" && exit 1",
    "dev": "cross-env PORT=8080 nodemon app.js",
    "prod": "cross-env PORT=80 node app.js"
},

Then, for example, use this command to run the application in a development environment:

npm run dev

However, this method of storing environment variables in the 'package.json' is generally not recommended. It is a much better practice to store it either with a library like 'dotenv' or in a 'config.js' file.
It is important to add the files that contain the environment variables to the .gitignore, so they do not get pushed to the repository. The 'package.json' needs to get pushed, so this is why it is a bad practice to have them in there.
Here is an example using 'dotenv': 

// .env file
API_KEY=mysecretkey

// In your Node.js code
const apiKey = process.env.API_KEY;

Here is an example using 'config.js': 

// config.js
export default {
    apiKey: 'mysecretkey',
    dbConnectionString: 'mongodb://localhost/mydb',
    // other configuration options
};

// In your Node.js code
import config from './config.mjs';
const apiKey = config.apiKey;
